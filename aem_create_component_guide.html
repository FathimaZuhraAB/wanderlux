Overview
What you are about to build
A "Card" component — a reusable UI block with a title, description, and a button. Editors configure it by filling in a dialog. We will create every file from scratch.
The 5 things every AEM component needs
Component folder
A folder at /apps/mysite/components/card/. The folder name = the component name.
.content.xml
Defines the component — its title, group, and parent. AEM reads this to register it.
Dialog XML
The form editors fill in — title, description, link. Saved as JCR properties on the content node.
Sling Model
A Java class that reads those JCR properties and exposes them as getter methods for the template.
HTL file
The HTML template that calls the model and renders the output on the page.

Step 2 of 10
Create .content.xml — the component definition
This file tells AEM what this component is — its display name, which browser group it belongs to, and which parent component it extends. Create it directly inside the card/ folder.
fileui.apps/…/components/card/.content.xml
in AEM JCRThis file IS the node at /apps/mysite/components/card
Exact file content — copy this exactly
.content.xml
<?xml version="1.0" encoding="UTF-8"?> <jcr:root xmlns:cq="http://www.day.com/jcr/cq/1.0" xmlns:jcr="http://www.jcp.org/jcr/1.0" xmlns:sling="http://sling.apache.org/jcr/sling/1.0" jcr:primaryType="cq:Component" jcr:title="Card" jcr:description="A card with title, description, and CTA button" componentGroup="My Site - Content" sling:resourceSuperType="core/wcm/components/teaser/v2/teaser" />
What every line does
jcr:primaryType
Always cq:Component — tells AEM this node is a component definition. Required.
jcr:title
"Card" — name shown in the component browser panel when editors drag components. Make it descriptive.
jcr:description
Tooltip shown when hovering the component in the browser. Helps editors understand what it does.
componentGroup
"My Site - Content" — which tab/group in the component browser. All your custom components should share the same group name.
sling:resourceSuperType
The parent component we inherit from. We extend Core Components' Teaser so we get image handling, policies, and accessibility for free. Can be removed if building from scratch.
After deployment, look for "My Site - Content" as a group in the component browser in the page editor. Your "Card" component will appear there.
Step 3 of 10
Create the dialog — the author form
The dialog is what pops up when an editor clicks the wrench (Configure) on your component. It is a single XML file inside the _cq_dialog subfolder. Each field saves a JCR property.
fileui.apps/…/components/card/_cq_dialog/.content.xml
this createsJCR node at /apps/mysite/components/card/_cq_dialog
Exact file content
_cq_dialog/.content.xml
<?xml version="1.0" encoding="UTF-8"?> <jcr:root xmlns:sling="http://sling.apache.org/jcr/sling/1.0" xmlns:granite="http://www.adobe.com/jcr/granite/1.0" xmlns:cq="http://www.day.com/jcr/cq/1.0" xmlns:jcr="http://www.jcp.org/jcr/1.0" xmlns:nt="http://www.jcp.org/jcr/nt/1.0" jcr:primaryType="nt:unstructured" sling:resourceType="cq/gui/components/authoring/dialog" jcr:title="Card Settings"> <content jcr:primaryType="nt:unstructured" sling:resourceType="granite/ui/components/coral/foundation/container"> <items jcr:primaryType="nt:unstructured"> <!-- Field 1: Title --> <title jcr:primaryType="nt:unstructured" sling:resourceType="granite/ui/components/coral/foundation/form/textfield" fieldLabel="Card Title" fieldDescription="The main heading of the card" name="./title" required="{Boolean}true"/> <!-- Field 2: Description --> <description jcr:primaryType="nt:unstructured" sling:resourceType="granite/ui/components/coral/foundation/form/textarea" fieldLabel="Description" fieldDescription="Supporting text below the title" name="./description" rows="3"/> <!-- Field 3: CTA Link --> <ctaLink jcr:primaryType="nt:unstructured" sling:resourceType="granite/ui/components/coral/foundation/form/pathfield" fieldLabel="Button Link" fieldDescription="Page or URL for the button" name="./ctaLink"/> <!-- Field 4: CTA Text --> <ctaText jcr:primaryType="nt:unstructured" sling:resourceType="granite/ui/components/coral/foundation/form/textfield" fieldLabel="Button Text" name="./ctaText"/> </items> </content> </jcr:root>
The critical rule — the name property
name="./title"
The ./ prefix means "relative to this component's content node". After saving, the value appears as a title property at /content/mysite/.../jcr:content/root/card
node name vs name
The XML node name (<title) is just for organization. The name attribute is what actually determines the JCR property name.
Step 4 of 10
Write the Sling Model — the Java data layer
The Sling Model reads the JCR properties that the editor set in the dialog, and exposes them as Java getter methods for the HTL template. Create TWO files — an interface and an implementation.
interfacecore/src/main/java/com/mysite/core/models/CardModel.java
implcore/src/main/java/com/mysite/core/models/impl/CardModelImpl.java
File 1 — CardModel.java (the interface)
CardModel.java
package com.mysite.core.models; public interface CardModel { String getTitle(); String getDescription(); String getCtaLink(); String getCtaText(); boolean isHasLink(); }
File 2 — CardModelImpl.java (the implementation)
CardModelImpl.java
package com.mysite.core.models.impl; import com.mysite.core.models.CardModel; import org.apache.sling.api.resource.Resource; import org.apache.sling.models.annotations.Model; import org.apache.sling.models.annotations.Default; import org.apache.sling.models.annotations.DefaultInjectionStrategy; import org.apache.sling.models.annotations.injectorspecific.ValueMapValue; import javax.annotation.PostConstruct; import org.apache.commons.lang3.StringUtils; @Model( adaptables = Resource.class, adapters = { CardModel.class }, resourceType = "mysite/components/card", defaultInjectionStrategy = DefaultInjectionStrategy.OPTIONAL ) public class CardModelImpl implements CardModel { // These read the JCR properties saved by the dialog @ValueMapValue private String title; @ValueMapValue private String description; @ValueMapValue private String ctaLink; @ValueMapValue @Default(values = "Read More") private String ctaText; // Computed fields — set in @PostConstruct private boolean hasLink; @PostConstruct protected void init() { // Runs after all @ValueMapValue fields are injected hasLink = StringUtils.isNotBlank(ctaLink); } @Override public String getTitle() { return title; } @Override public String getDescription() { return description; } @Override public String getCtaLink() { return ctaLink; } @Override public String getCtaText() { return ctaText; } @Override public boolean isHasLink() { return hasLink; } }
What each annotation does
@Model
resourceType must exactly match the component's folder path relative to /apps. This links the model to the component.
@ValueMapValue
Reads the JCR property with the same name as the field. private String title reads the title property saved by the dialog's name="./title" field.
@Default
Fallback value if the editor left the field blank. ctaText defaults to "Read More" if empty.
@PostConstruct
Runs after all injections. Use it for any computed logic — in this case, checking if ctaLink is not blank.
Also add this to core/pom.xml if not already present
core/pom.xml — bnd plugin config
<!-- Inside the bnd-maven-plugin configuration: --> <bnd> Sling-Model-Packages: com.mysite.core.models </bnd>
Without Sling-Model-Packages in the bnd config, AEM will never scan your models package and data-sly-use will return null silently.
Step 5 of 10
Write the HTL template — card.html
The HTL file produces the HTML output. It uses data-sly-use to load the Sling Model, then outputs its values. The filename MUST be card.html — matching the component folder name.
fileui.apps/…/components/card/card.html
naming ruleMUST match the folder name — folder is "card" so file must be "card.html"
Exact file content
card.html
<!-- Load the Sling Model. "model" is now your Java object. --> <div data-sly-use.model="com.mysite.core.models.CardModel" class="card"> <!-- Card title — only render if title is not null --> <h3 class="card__title" data-sly-test="${model.title}"> ${model.title} </h3> <!-- Description — only render if description is not null --> <p class="card__description" data-sly-test="${model.description}"> ${model.description} </p> <!-- CTA Button — only render if a link was set --> <a class="card__cta" href="${model.ctaLink @ context='uri'}" data-sly-test="${model.hasLink}"> ${model.ctaText} </a> </div>
Key HTL concepts used here
data-sly-use.model
Loads CardModel and binds it to the variable model. Every ${model.xxx} call goes to the Java getter getXxx().
${model.title}
Calls getTitle() in Java and outputs the result. Automatically HTML-escaped for security — no XSS risk.
data-sly-test
Hides the element entirely if the expression is falsy (null, empty, false). The element is not even in the DOM output.
@ context='uri'
Tells HTL this value is a URL — apply URL encoding instead of HTML encoding. Always use this on href and src attributes.
model.hasLink
Calls isHasLink() in Java (boolean getters use "is" prefix). HTL removes the "is" automatically.

Step 7 of 10
Deploy to AEM
Now that all files are created, deploy everything to your local AEM instance with one Maven command. AEM must be running on port 4502.
Run this command from the project root
terminal — project root folder
# Deploy everything (first time or when unsure) mvn clean install -PautoInstallPackage # Or if you only changed Java files (faster) mvn clean install -PautoInstallBundle -pl core # Or if you only changed HTL/dialog/XML files (faster) mvn clean install -PautoInstallPackage -pl ui.apps
Verify the deployment worked — check these 3 things
1. Bundle active?
Go to http://localhost:4502/system/console/bundles → search mysite.core → must say Active. If it says "Installed", your Java has a broken import.
2. Component in JCR?
Go to CRXDE at http://localhost:4502/crx/de → navigate to /apps/mysite/components/card → you should see the folder and files.
3. Model registered?
Go to /system/console/services → search CardModel → it should appear in the list.
Common errors and fixes
BUILD FAILURE
Maven compilation error — read the error message. Usually a Java syntax error or missing import.
Bundle "Installed"
A Java package import in your code doesn't exist. Check the Import-Package section in /system/console/bundles → click your bundle → look for "Missing Dependencies".
Component missing in AEM
ui.apps didn't deploy. Run mvn clean install -PautoInstallPackage -pl ui.apps separately.
Step 8 of 10
Allow the component on a template
Before editors can drag the Card component onto a page, a template author must add it to the template's allowed components policy. This is done entirely in the AEM UI — no code needed.
open this URLhttp://localhost:4502/libs/wcm/core/content/sites/templates.html/conf/mysite
Steps in AEM UI
1
Open the Templates console. Find your template (e.g. "Article Page") → click it to open the template editor.
2
Click on the Layout Container in the template → the blue toolbar appears. Click the Policy icon (looks like a shield/clipboard).
3
In the policy editor → scroll to "Allowed Components". Find your group "My Site - Content" → expand it → check the box next to Card.
4
Click the checkmark to save. The Card component is now available on all pages using this template.
If you do not see "My Site - Content" in the allowed components list, your componentGroup in .content.xml does not match. Check it — it is case-sensitive.
